# Workflows

## All Tasks Which Can Be Completed

Tasks Seller can complete:

- Login or register
- Create a bundle
- View forecast of bundle &rarr; predicted reservations and expected no-show probability, as well as recommendations
- Post bundle
- View all posted bundles, past and present
- Edit posted bundles
- Filter to find bundles &rarr; date, category or status (Including “All” for specific filters)
- View all bundle reservations
- Update status of posted bundle to &rarr; "Preparing" when created, "Active" during pick-up window" and "Expired" when pick-up window is done
- Update status of reserved bundles &rarr; "Reserved" when not collected yet, "Collected" when successfully collected, "No-show" if not collected
- See reports associated with bundles
- Respond to reports
- Highlight reports to Developers &rarr; done with catagories such as Seller and Developer related reports
- Be notified of new reports leading back to bundle it belongs to in Reports
- Find bundles from collection code through search
- Validate collections through collection code
- Filter through reserved bundles by date, category, bundle status or reservation/collection status
- Make report to developers
- View forecast analytics
- View action log
- Filter through action log by date, time &rarr; See compact list of newest to oldest actions taken
- Edit company details

Tasks a Consumer can complete:

- Login/register
- Scroll main page of bundles
- Filter bundles &rarr; category, time-slot, “without certain allergy” and location
- Click on "View" on bundle and see more information &rarr; Bundle information including bundle description as well as a sidebar with the company information (Name, location, open-hours etc.)
- Add report to bundle (open pop-up) &rarr; for collection preferences or notes (accessibility)
- Reserve bundle, report sent when reserved (Make this clear)
- Receive collection code (pop-up with reservation details)
- Get notification on profile directing to reserved bundles list
- View reserved bundle list
- Filter through past and present reserved bundles, defaulted on today and uncollected bundles
- View collection code on reserved uncollected bundle
- Make report to bundle from list
- Get notification on profile if streak is updated
- View streak on profile
- Streak highlighted if streak updated
- See number representation of streak
- See bundle status through reserved bundle list
- Be notified of bundle bundle status (active/"ready-to-collect")
- Receive badges
- Recieve notification of new badges
- View badges on profile
- See impact report page
- See small stats from report on profile (e.g. 15 meals saved so far, CO2 saved)
- See more detailed impact report on dedicated page
- Change accessibility settings (dark mode, dyslexia friendly font, aria labels, the ability to navigate it via a keyboard, an easily readable contrast between elements
- Edit personal details

Tasks a Developer can complete (Incomplete):

- Login
- See all Sellers
- See all Seller reports
- Enter Seller profile
- Developer view of profile
- See Seller View of profile
- See Consumer reports of Seller
- Reply to reports from Seller
- Reply to reports from Consumer of Seller?

## Sprint 1 --

seller posts bundle &rarr; consumer reserves &rarr; claim code
generated &rarr; seller marks collected &rarr; consumer streak updates.

Seller:

- Logs-in or register
- On Seller's dashboard
- Clicks create bundle
- Creates bundle (Fill in essential fields, category, pick-up time)
- Finalize Bundle - Click complete, see summary of info and forecast
- Click Edit to correct anything
- Click Post to post the bundle
- Click Discard to discard progress
- Is back to empty bundle post
- Clicks on Bundle Posts
- See List of bundle posts &rarr; Buttons/options on bundles: Edit, Delete, status - for manual changes to status (Note: All bundle status changes are done automatically at changes in time)
- Able to filter bundle options: Day (Today, yesterday, 1 week, 1 month), status, catagory
- Clicks on Reserved Bundles
- See Bundle Reservations &rarr; Buttons/options on reservations: reservation/collection status, verify code (Give confirmation and confirmation bottum is made clickable), Link to reports on bundle
- Click Reports on bundle
- View bundle reports &rarr; clicks back and is returned to original place in posted bundles
- Filter reserved bundles &rarr; date (default on today), bundle status (Status of the bundle it belongs to, default on "All"), reservation/collection (default on "reserved")
- Search reserved bundles by collection code (In filters section)
- Clicks Reports &rarr; Sees all reports to all bundles in compact list

Consumer

- Logs-in or register
- On main page
- Scrolls the main page or searches the main page with filters
- See “Filter” bar on side or at top &rarr; choose filters like price, company, location, catagory, "without certain allergy"
- Clicks on “View” for desired bundle - See more info (Click "back" or an arrow to return to the main page)
- Click "Add Report" to add in accessibility requests to reservation
- Select quantity of bundle to reserve
- Clicks on "Reserve"
- Receives pop-up with bundle details and pick-up code and taken back to main page
- Notification on user profile indicating where the bundle information can be re-accessed
- Clicks user profile
- Clicks “Reserved Bundles” to see pending bundles &rarr; can see code on preview of bundle.
- Click link on bundle to make report
- Click on "Cancel" to cancel reservation of bundle
- Click on "Impact Report" to see more detailed impact report data
